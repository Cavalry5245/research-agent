目标：

- 默认在 conda 环境中执行项目命令
- 项目内已存在 `.env`、样例 PDF、parsed metadata、基础测试与文档

## 一、必须优先确认的事项

### 1. benchmark 样本策略

### 3. 真实模型评估策略
当前可选方案：

- [ ] 直接使用当前 `.env` 配置跑真实链路评估
- [ ] 先只做离线 / rule-based / mock 骨架，不做真实评估
- [ ] 先尝试真实链路，失败后自动降级到 mock / offline

说明：
- 当前项目中 `.env` 已存在，且关键字段已配置

建议：
- 若你想兼顾推进速度与真实性，优先选“先尝试真实链路，失败后自动降级”
